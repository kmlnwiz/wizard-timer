# WIZARD TIMER

周回タイム計測に特化したストップウォッチ／ラップ計測アプリです。金曜16:00〜月曜16:00の72時間イベントを想定し、ラップの記録・統計・時速（周回数／ポイント）の可視化を行います。

## 主な機能

- **ストップウォッチ / ラップ計測**
  - 開始・停止・再開（経過秒数が残っている場合は「再開」表示）
  - 「0秒から」スタート（履歴は残したまま新しい計測セッションを開始）
  - 現在ラップの秒数リセット
  - ラップ記録（3秒クールダウンで連打を防止）
  - **リロードしても計測中なら継続**（`localStorage` に状態を保存し、経過時間を再計算して復元）
- **時計パネル** — 現在時刻・金曜16:00からの経過・月曜16:00までのカウントダウンを秒まで並列表示
- **ラップ統計** — 直近1時間 / 前の1時間（平均ラップ・最速ラップ・時速）と、全体統計（平均ラップ・最速ラップ・平均時速・最高時速）
- **サマリ** — 合計時間・合計ラップ数・平均ラップ・最速ラップ・平均時速・最高時速・理論時速、時間枠ごとの周回数／平均ラップの複合グラフ、画像（PNG）保存
- **時速表示** — 1周あたりのポイントを設定すると `pt/h`、未設定時は `周/h`
- **ラップ編集** — ラップの分割・削除（後続への時間合算あり／完全削除）
- **データ管理** — CSV ダウンロード、全履歴リセット（リセット前の CSV 保存を案内）
- **キーバインド** — ラップ / 開始・停止 / 0秒から開始 に任意キーを割り当て
- **ダークモード** — システム / ライト / ダーク

## データの保存

すべてのデータはブラウザの `localStorage` に保存されます（サーバー送信なし）。

| キー | 内容 |
|---|---|
| `wizard-timer:laps` | ラップ履歴 |
| `wizard-timer:settings` | ポイント・キーバインド設定 |
| `wizard-timer:dark-mode` | ダークモード設定 |
| `wizard-timer:stopwatch` | ストップウォッチの状態（リロード継続用） |

## 集計の考え方

- イベントは直近の金曜16:00を起点とした72時間。
- 実測枠は `NN:55` 〜 次の `NN:55` の1時間単位。
- 各枠の周回数がその時間の「時速」に相当し、1周あたりのポイントを設定するとポイント時速（`pt/h`）に換算されます。
- 理論時速は、最速ラップのペースで1時間回り続けた場合の周回数から算出します。
